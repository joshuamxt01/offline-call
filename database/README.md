# database/

Canonical PostgreSQL data model for Nexa (Neon-compatible).

- **`schema.sql`** — human-readable canonical DDL. Mirrors `backend/src/db/schema.ts` (Drizzle).
- **`migrations/`** — ordered SQL migrations generated by Drizzle (`backend` owns generation).
- **`seeds/seed.sql`** — quick psql seed (placeholder password hash).

## Apply schema

**Preferred (from backend, hashes seed passwords properly):**
```bash
cd ../backend
npm run migrate    # applies Drizzle migrations to $DATABASE_URL
npm run seed       # inserts admin/alice/bob with real Argon2id hashes
```

**Raw psql (local smoke test):**
```bash
psql "$DATABASE_URL" -f schema.sql
psql "$DATABASE_URL" -f seeds/seed.sql   # placeholder hashes — cannot log in
```

## Two stores
Durable data lives here (Postgres). Ephemeral, high-churn state (presence, live call
state, rate limits) lives in Upstash Redis — see `docs/02-DATABASE-SCHEMA.md`.
